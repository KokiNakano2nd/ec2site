## 改善項目一覧

### 優先度：高（P1）

#### 1. 商品検索・カテゴリフィルター
- **概要**: キーワード検索バーとカテゴリタブによる絞り込み
- **理由**: 商品数が増えると必須。UXの根幹
- **対応範囲**:
  - Frontend: ヘッダーに検索入力 or 商品一覧上部に検索バー
  - Backend: `/products?q=keyword&category=xxx` クエリパラメータ対応
- **工数目安**: 1〜2日

#### 2. ページネーション / 無限スクロール
- **概要**: 商品一覧を一定件数ずつ分割表示
- **理由**: 商品が増えたときのパフォーマンス対策
- **対応範囲**:
  - Frontend: ページ送りボタン or スクロール検知
  - Backend: `/products?page=1&limit=12` 対応
- **工数目安**: 1日

#### 3. 在庫切れ表示の強化
- **概要**: 在庫0商品の「在庫切れ」バッジ表示、カートへの追加不可
- **理由**: 現状は低在庫バッジのみ。在庫0時の制御が不十分
- **対応範囲**: Frontend のみ
- **工数目安**: 半日

---

### 優先度：中（P2）

#### 4. レビュー・評価機能（★）
- **概要**: 商品に対して星1〜5の評価とコメントを投稿
- **理由**: Amazonらしさの核心。購買判断の材料になる
- **対応範囲**:
  - Frontend: 商品詳細にレビュー一覧・投稿フォーム
  - Backend: `reviews` テーブル、POST `/products/{id}/reviews`
- **工数目安**: 2〜3日

#### 5. お気に入り（ウィッシュリスト）
- **概要**: 商品をお気に入り登録して後で見返せる
- **理由**: ユーザーの回遊率・再訪率向上
- **対応範囲**:
  - Frontend: 商品カード・詳細にハートアイコン
  - Backend: `favorites` テーブル
- **工数目安**: 1〜2日

#### 6. 注文ステータス管理の強化
- **概要**: 注文を「受付中 → 処理中 → 発送済み → 完了」と段階管理
- **理由**: 現状はステータスが固定。注文追跡の体験を改善
- **対応範囲**:
  - Frontend: 注文履歴にステータスバー表示
  - Backend: ステータス更新APIの拡張
  - Admin: 注文管理画面でステータス変更操作
- **工数目安**: 1〜2日

#### 7. クーポン・割引コード
- **概要**: カート画面でクーポンコードを入力し割引を適用
- **理由**: 購買促進・販促キャンペーンへの対応
- **対応範囲**:
  - Frontend: カート画面にクーポン入力欄
  - Backend: `coupons` テーブル、検証API
- **工数目安**: 2日

---

### 優先度：低（P3）

#### 8. レコメンド機能
- **概要**: 「この商品を見た人はこちらも」セクション
- **理由**: Amazonらしさの演出。ただし本格的なロジックは複雑
- **対応範囲**:
  - Backend: 同カテゴリ商品をランダム or 閲覧数順で返すシンプル実装から始める
- **工数目安**: 1〜2日（シンプル版）

#### 9. 画像複数枚対応
- **概要**: 商品に複数画像を登録し、詳細画面でサムネイル切り替え
- **理由**: 商品の見せ方を強化するが、画像ストレージ設計が必要
- **対応範囲**: Frontend + Backend + ストレージ設計
- **工数目安**: 3日〜

#### 10. メール通知
- **概要**: 注文確認・発送通知などをメール送信
- **理由**: ユーザー体験向上。ただし外部サービス連携が必要
- **対応範囲**: Backend（SendGrid / SES等）
- **工数目安**: 2〜3日

## 適用計画（フェーズ別）

### Phase 1：基盤強化（〜2週間）

商品が増えても崩れないUXの土台を作る。

```
Week 1: 在庫切れ表示強化 → 検索・フィルター実装
Week 2: ページネーション → 注文ステータス強化
```

### Phase 2：エンゲージメント向上（〜1ヶ月）

ユーザーが「また来たい」と思える機能を追加。

```
Week 3: お気に入り機能
Week 4: レビュー・評価機能
```

### Phase 3：収益・販促機能（〜1.5ヶ月）

売上につながる機能を追加。

```
Week 5: クーポン・割引コード
Week 6: レコメンド機能（シンプル版）
```

### Phase 4：コンテンツ強化（優先度に応じて）

見た目・体験のブラッシュアップ。

```
画像複数枚対応
メール通知
その他UX改善
```

## 技術メモ

- **Backend**: FastAPI + SQLite（現状）。P2以降でテーブル追加が増えるため、マイグレーション管理（Alembic等）の導入を検討
- **Frontend**: React + Vite。状態管理は現状のuseState/Context で対応可能。P2以降で複雑になればZustand等を検討
- **画像**: 現状はURL文字列のみ。複数枚対応時はローカルストレージ or S3等を要検討
