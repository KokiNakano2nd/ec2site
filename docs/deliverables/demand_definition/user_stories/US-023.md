# US-023: 返品を承認・却下する(管理者)

[← User Story一覧に戻る](../02_user_stories.md)

**業務領域**: 注文管理業務(管理者向け)

**As a** 管理者
**I want** 顧客からの返品申請を承認または却下したい
**So that** 現物の返送状況等を確認したうえで、適切な場合のみ返金・在庫戻しを行える

**関連する業務フロー図ステップ**: [[../business_flow/08_order_admin|08_order_admin.md]]「業務フロー図(To-Be): 返品承認・却下」

**複雑度判定**: 複雑(分岐条件1: 承認/却下で処理が分岐。承認時はStripe決済済みかどうかで返金処理の要否が分かれ、在庫・クーポン使用回数のロールバックを伴うため、ユースケース化(UC-008)する)

**Confirmation(受け入れ条件)**:

```
Given 管理者がログイン済みであり、返品申請中の注文が存在する
When 注文管理画面で当該注文を「承認」する
Then 注文が返品完了になり、在庫が注文数量分戻り、クーポンを使用していた場合は使用回数が戻り、Stripe決済済みの場合は返金され、顧客に通知メールが送信される

Given 管理者がログイン済みであり、返品申請中の注文が存在する
When 注文管理画面で当該注文を「却下」する
Then 注文が発送済みステータスに戻り、在庫・クーポン使用回数・返金は変更されず、顧客に通知メールが送信される
```
