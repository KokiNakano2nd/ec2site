# US-020: 退会する

[← User Story一覧に戻る](../02_user_stories.md)

**業務領域**: 会員管理業務

**As a** 顧客
**I want** アカウントを退会(削除)したい
**So that** サービスの利用をやめ、個人情報の保持を最小限にできる

**関連する業務フロー図ステップ**: [[../business_flow/02_membership|02_membership.md]]「業務フロー図(To-Be): 退会」

**複雑度判定**: 複雑(分岐条件1: パスワード不一致。加えて、注文履歴等の関連データの扱いを決める必要があり、単純なCRUDより業務判断が伴うため、ユースケース化(UC-005)する)

**Confirmation(受け入れ条件)**:

```
Given 顧客がログイン済みである
When プロフィール画面で「退会する」を選択し、現在のパスワードを正しく入力する
Then アカウントが論理削除され、メールアドレス等の個人情報が匿名化され、ログアウトされる

Given 顧客がログイン済みである
When プロフィール画面で「退会する」を選択し、誤ったパスワードを入力する
Then 「パスワードが正しくありません」エラーが表示され、アカウントは削除されない

Given 顧客が退会済みである
When 退会前に行った注文の履歴を運営(管理者)が確認する
Then 注文記録(注文明細・合計金額等)は削除されず、業務記録として参照できる(NFR-013参照)
```
