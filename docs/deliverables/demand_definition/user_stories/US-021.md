# US-021: 注文をキャンセルする

[← User Story一覧に戻る](../02_user_stories.md)

**業務領域**: 商品購入業務

**As a** 顧客
**I want** 発送前の注文をキャンセルしたい
**So that** 注文内容を誤った場合や不要になった場合に、発送されてしまう前に取り消せる

**関連する業務フロー図ステップ**: [[../business_flow/01_product_purchase|01_product_purchase.md]]「業務フロー図(To-Be): 注文キャンセル・返品申請」

**複雑度判定**: 複雑(分岐条件2: 対象注文のステータスが受付中/処理中以外の場合、Stripe決済済みかどうかで返金処理の要否が分かれる。加えて在庫・クーポン使用回数のロールバックを伴うため、ユースケース化(UC-006)する)

**Confirmation(受け入れ条件)**:

```
Given 顧客がログイン済みであり、自分の注文(ステータス: 受付中または処理中)が存在する
When 注文履歴画面で当該注文の「キャンセルする」を選択する
Then 注文がキャンセル済みになり、在庫が注文数量分戻り、クーポンを使用していた場合は使用回数が戻り、Stripe決済済みの場合は返金される

Given 顧客がログイン済みであり、自分の注文(ステータス: 発送済み以降)が存在する
When 注文履歴画面で当該注文の「キャンセルする」を選択する
Then 「発送済みの注文はキャンセルできません」エラーが表示され、注文は変更されない
```
