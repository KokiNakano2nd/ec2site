# US-026: パスワードをリセットする

[← User Story一覧に戻る](../02_user_stories.md)

**業務領域**: 会員管理業務

**As a** 顧客
**I want** パスワードを忘れた場合にメール経由で再設定したい
**So that** アカウントにログインできなくなった場合でも、自力でログインを回復できる

**関連する業務フロー図ステップ**: [[../business_flow/02_membership|02_membership.md]]「業務フロー図(To-Be): パスワードリセット」

**複雑度判定**: 複雑(分岐条件: リセットトークンの有効・無効/期限切れ。加えて、メールアドレスの存在有無を顧客に開示しないためのセキュリティ上の考慮が必要であり、単純なCRUDより業務判断が伴うため、ユースケース化(UC-009)する)

**Confirmation(受け入れ条件)**:

```
Given 顧客がログイン画面にいる
When 「パスワードをお忘れですか?」を選択し、登録済みのメールアドレスを入力する
Then リセットリンク付きのメールが送信され、「メールを送信しました」という同一の完了画面が表示される

Given 顧客がログイン画面にいる
When 「パスワードをお忘れですか?」を選択し、登録されていないメールアドレスを入力する
Then メールは送信されないが、登録済みの場合と同一の「メールを送信しました」という完了画面が表示される(メールアドレスの存在を顧客に開示しない)

Given 顧客が有効なリセットリンクを開いている(発行から24時間以内、未使用)
When 新しいパスワードを入力し、確定する
Then パスワードが更新され、そのリセットトークンは再利用できなくなり、ログイン画面に遷移する

Given 顧客が期限切れ、または既に使用済みのリセットリンクを開いている
When 新しいパスワードを入力し、確定する
Then 「リンクが無効です。再度お手続きください」エラーが表示され、パスワードは更新されない
```
