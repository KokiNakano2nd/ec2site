# US-031: 自分の注文履歴・詳細を確認する

[← User Story一覧に戻る](../02_user_stories.md)

**業務領域**: 商品購入業務

**As a** ログイン済み顧客

**I want** 自分の注文履歴と各注文の購入内容を確認したい

**So that** 購入内容と現在の注文状態を把握できる

**関連する業務フロー図ステップ**: [[../business_flow/01_product_purchase|01_product_purchase.md]]「注文キャンセル・返品申請」の注文選択ステップ

**複雑度判定**: 単純(本人の注文だけを作成日時降順で参照する処理であり、状態変更を伴わないため)

**Confirmation(受け入れ条件)**:

```text
Given 顧客がログイン済みで、自分の注文が存在する
When 顧客が注文履歴を開く
Then 自分の注文だけが作成日時の新しい順に表示される

Given 顧客がログイン済みで、自分の注文が存在する
When 顧客が注文履歴を開く
Then 各注文に注文明細・金額・状態等の購入内容が表示される

Given 存在しない注文、または他の顧客の注文IDが指定される
When 顧客が注文詳細を取得する
Then 注文が存在しないことを示すエラーが返り、他者の注文情報は開示されない
```
