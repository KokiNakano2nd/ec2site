# US-027: メールアドレスの本人確認を行う

[← User Story一覧に戻る](../02_user_stories.md)

**業務領域**: 会員管理業務

**As a** 顧客
**I want** 登録したメールアドレスが本人の受信できるものであることを確認したい(または運営として確認させたい)
**So that** 他人のメールアドレスの誤登録・悪用を防ぎ、パスワードリセット等の重要な通知が確実に本人に届くようにできる

**関連する業務フロー図ステップ**: [[../business_flow/02_membership|02_membership.md]]「業務フロー図(To-Be): メールアドレスの本人確認」

**複雑度判定**: 複雑(分岐条件: 確認用トークンの有効・無効/期限切れ。加えて、確認メールの再送機能もあり、単純なCRUDより業務判断が伴うため、ユースケース化(UC-010)する)

**Confirmation(受け入れ条件)**:

```
Given 顧客が新規登録を行った直後である
When 登録処理が成功する
Then 入力されたメールアドレス宛に確認リンク付きのメールが送信され、アカウントは未確認状態(is_verified=false)で作成される

Given 顧客が有効な確認リンクを開いている(発行から7日以内、未使用)
When 確認リンクを開く
Then アカウントが確認済み状態(is_verified=true)に更新され、「メールアドレスを確認しました」と表示される

Given 顧客が期限切れ、または既に使用済みの確認リンクを開いている
When 確認リンクを開く
Then 「リンクが無効です。再度お手続きください」エラーが表示され、確認状態は更新されない

Given 顧客がログイン済みで、メールアドレスが未確認である
When プロフィール画面で「確認メールを再送する」を選択する
Then 新しい確認リンク付きのメールが再送される

Given 顧客のメールアドレスが未確認である
When 通常通りログイン・商品購入等の既存機能を利用する
Then いずれの機能も制限されず利用できる(非ブロッキング設計。プロフィール画面に未確認バッジが表示されるのみ)
```
