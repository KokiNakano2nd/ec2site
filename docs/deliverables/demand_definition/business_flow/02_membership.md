# 業務フロー図: 会員管理業務

[← 業務フロー図一覧に戻る](../01_business_flow.md) / 全体ルール: [[../../../README|docs/README.md]]

### 概要

顧客がアカウントを登録し、ログインしてシステムを利用できるようにする業務。

### 登場アクター

- 顧客
- システム(EC_SITE)

### 業務フロー図(As-Is)

該当なし。本機能(会員登録・ログイン)は、本ECサイトの導入によって新たに生まれた業務であり、対応する紙・電話ベースのAs-Is業務フローは存在しない(既存の商品購入業務のAs-Isは、顧客を識別せず都度電話・FAXで注文を受け付ける方式だったため、「会員」という概念自体が存在しなかった)。

### 課題・問題点

該当なし(As-Is業務が存在しないため)。

### 業務フロー図(To-Be)

```mermaid
flowchart TD
    subgraph 顧客
        A((●)) --> B["メールアドレス・パスワードを入力し新規登録する<br/><<システム化>>"]
        B --> E["メールアドレス・パスワードを入力しログインする<br/><<システム化>>"]
    end
    subgraph システム
        C{"メールアドレスが登録済みか<br/><<システム化>>"}
        D["パスワードをハッシュ化しアカウントを作成する<br/><<システム化>>"]
        F{"メールアドレス・パスワードが一致するか<br/><<システム化>>"}
        G["アクセストークン(JWT)を発行する<br/><<システム化>>"]
    end
    B --> C
    C -->|登録済み| H["「このメールアドレスは既に登録されています」を表示する"]
    C -->|未登録| D
    D --> E
    E --> F
    F -->|不一致| I["「メールアドレスまたはパスワードが正しくありません」を表示する"]
    F -->|一致| G
    G --> J((◎))
    H --> J
    I --> J
```
