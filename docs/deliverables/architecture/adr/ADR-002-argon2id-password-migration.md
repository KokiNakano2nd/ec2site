# ADR-002: bcryptからArgon2idへログイン時段階移行する

- **状態**: Accepted
- **決定日**: 2026-07-13
- **決定者**: プロジェクト所有者

## コンテキスト

passlibの保守停止とbcrypt新バージョンとの非互換を解消した後、新規パスワードの保護方式と既存bcryptハッシュの移行方法を決める必要があった。

## 検討した選択肢

- bcryptを継続: 既存互換性は高いが、新規保存方式として優先しない
- 全ユーザーを一括リセット: 実装は単純だが利用者負担が大きい
- ログイン成功時段階移行: 平文を知る成功時だけ安全に再ハッシュできる

## 決定

新規登録・リセットはArgon2idを使用する。既存bcryptは検証専用とし、ログイン成功時にArgon2idへ再ハッシュする。Argon2パラメーターが更新された場合も`check_needs_rehash`で段階移行する。

## 影響

- bcrypt依存は既存ハッシュ検証が不要になるまで残る
- ログインしないユーザーのハッシュは移行されない
- ハッシュ形式と移行動作を認証テストで維持する
