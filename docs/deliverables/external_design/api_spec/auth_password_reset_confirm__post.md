# POST /auth/password-reset/confirm — パスワードを再設定する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 会員管理業務 / **リソース**: /auth

**概要**: リセットトークンを検証し、新しいパスワードを設定する(S-009)
**元になった機能**: F-036
**認証**: 不要(リセットトークン自体が本人確認の代わりとなる)

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| token | string | 必須 | リセットリンクに含まれるトークン |
| new_password | string | 必須 | 新しいパスワード |

**レスポンス(200)**: 本文なし

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 400 | トークンが存在しない、既に使用済み、または発行から24時間を超えて期限切れ | 「リンクが無効です。再度お手続きください」 |

**実装上の注記**: 成功時、新しいパスワードをハッシュ化して`hashed_password`に保存し、`password_reset_token`・`password_reset_token_expires_at`を`NULL`にクリアすることでトークンを再利用不能にする(1回限りの使用制限、UC-009参照)。パスワードリセット後も、それ以前に発行済みのJWTアクセストークンは個別には失効しない(本システムのJWTはステートレスであり、トークン単位の失効の仕組みを持たないため。将来的な課題)
