# DELETE /users/me — 退会する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 会員管理業務 / **リソース**: /users

**概要**: ログイン中のアカウントを退会(論理削除+個人情報の匿名化)する(S-007)
**元になった機能**: F-030
**認証**: 必要(Bearerトークン)

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| password | string | 必須 | 本人確認のための現在のパスワード |

**レスポンス(204)**: 本文なし

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 403 | パスワードが一致しない | 「パスワードが正しくありません」 |
| 401 | 未ログイン、または既に退会済み(トークンのユーザーが`is_active=false`) | 「認証情報が無効です」 |

**実装上の注記**: 退会後、当該ユーザーのメールアドレスは`deleted-user-{id}@deleted.invalid`形式に匿名化されるため、同一メールアドレスでの再登録(`POST /auth/register`)が可能になる。配送先(`addresses`)・お気に入り(`favorites`)は削除されるが、注文履歴(`orders`/`order_items`)・レビュー(`reviews`)は削除されない(UC-005参照)
