# GET /orders — 注文履歴を取得する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 商品購入業務 / **リソース**: /orders

**概要**: 注文履歴画面(S-004)の表示データを取得する。ログイン中ユーザー自身の注文のみ、作成日時降順で返す
**元になった機能**: F-006, F-008(注文確定後の履歴表示)
**認証**: 必要(Bearerトークン)

**リクエスト**: なし

**レスポンス(200)**: `POST /orders`のレスポンスと同一形式の配列

**エラーレスポンス**: なし
