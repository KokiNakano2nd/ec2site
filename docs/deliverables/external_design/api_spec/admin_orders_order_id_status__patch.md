# PATCH /admin/orders/{order_id}/status — 注文ステータスを更新する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 注文管理業務(管理者向け) / **リソース**: /admin/orders

**認証**: 必要(Bearerトークン + 管理者権限)

**概要**: 注文のステータスを更新し、顧客にステータス変更通知メールを送信する(S-103)
**元になった機能**: F-027
**関連する通知**: `04_notification_design.md` N-002

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| order_id | integer(パスパラメータ) | 必須 | 注文ID |
| status | string | 必須 | 変更後のステータス。通常進行は`pending → processing → shipped → completed`。`pending → shipped`も許可する |

**レスポンス(200)**: 注文オブジェクト

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 404 | 注文が存在しない | 「注文が見つかりません」 |
| 400 | 未定義のステータス | 「無効な注文ステータスです」 |
| 400 | 許可されていない状態遷移 | 「許可されていない注文ステータス遷移です」 |
