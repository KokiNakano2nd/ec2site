# POST /auth/password-reset/request — パスワードリセットを要求する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 会員管理業務 / **リソース**: /auth

**概要**: 入力されたメールアドレス宛にパスワードリセットリンクを送信する(S-008)
**元になった機能**: F-036
**認証**: 不要

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| email | string | 必須 | メールアドレス |

**レスポンス(200)**: 本文なし(常に同一のレスポンス)

**エラーレスポンス**: なし(入力されたメールアドレスに一致する有効なアカウントが存在しない場合も、200を返しメールは送信しない。ユーザー列挙攻撃を防ぐための意図的な設計、UC-009参照)

**実装上の注記**: 一致する有効なアカウント(`is_active=true`)が見つかった場合、ランダムなリセットトークンを生成し`password_reset_token`・`password_reset_token_expires_at`(発行から24時間後)に保存したうえで、`{FRONTEND_URL}/reset-password?token=...`形式のリンクを含むメールを送信する(N-005)。メール送信に失敗した場合もエラーレスポンスは返さず、ERRORログを出力するのみ(既存のN-001〜N-004と同じパターン)
