# POST /auth/verify-email/resend — 確認メールを再送する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 会員管理業務 / **リソース**: /auth

**概要**: ログイン中のユーザーのメールアドレス宛に、確認用リンクを含むメールを再送する(S-007)
**元になった機能**: F-037
**認証**: 必要(Bearerトークン)

**リクエスト**: なし

**レスポンス(200)**: 本文なし

**エラーレスポンス**: なし(既に確認済み(`is_verified=true`)のユーザーが呼び出した場合も、エラーにはせず新しい確認メールを送信する。冪等な操作として扱う)

**実装上の注記**: 呼び出しのたびに新しい確認用トークンを生成し、既存のトークン(未使用の場合)を上書きする。そのため、古い確認リンクを複数発行していた場合、最後に送信されたリンクのみが有効になる
