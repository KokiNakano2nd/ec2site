# POST /orders — カード決済を使わずに注文を確定する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 商品購入業務 / **リソース**: /orders

**概要**: カート内商品から即時に注文を確定する(代金引換・銀行振込等を想定した非Stripe経路)(UC-003)
**元になった機能**: F-008
**認証**: 必要(Bearerトークン)

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| coupon_code | string \| null | 任意 | 適用するクーポンコード |

**レスポンス(201)**:
| 項目名 | 型 | 説明 |
|---|---|---|
| id | integer | 注文ID |
| total_price | float | 合計金額(税込) |
| discount_amount | float | 割引額 |
| coupon_code | string \| null | 適用されたクーポンコード |
| status | string | 注文状態(`pending`) |
| created_at | datetime | 作成日時 |
| items | array | 注文明細(`id`, `quantity`, `price`, `product`) |
| user_email | string \| null | 注文者のメールアドレス |

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 400 | カートが空 | 「カートが空です」 |
| 400 | カート内商品の在庫が不足 | 「{商品名}の在庫数が不足しています」 |
| 400 | クーポンコードが無効 | 「無効なクーポンコードです」 |
| 400 | クーポンの使用回数が上限に達している | 「このクーポンは使用回数の上限に達しています」 |
