# POST /auth/register — 会員登録する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 会員管理業務 / **リソース**: /auth

**概要**: メールアドレス・パスワードでアカウントを作成する(S-005)
**元になった機能**: F-009
**認証**: 不要

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| email | string | 必須 | メールアドレス |
| password | string | 必須 | パスワード(平文で送信し、サーバー側でハッシュ化) |

**レスポンス(201)**:
| 項目名 | 型 | 説明 |
|---|---|---|
| id | integer | ユーザーID |
| email | string | メールアドレス |
| is_admin | boolean | 管理者フラグ |
| is_verified | boolean | メールアドレス確認済みフラグ(2026-07-13追加、常に`false`で作成される) |
| created_at | datetime | 登録日時 |

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 400 | メールアドレスが既に登録済み | 「このメールアドレスは既に登録されています」 |
| 429 | 同一IPアドレスから1時間に5回を超えるリクエスト(NFR-022、2026-07-13追加) | 「リクエストが多すぎます。しばらくしてから再度お試しください」 |

**実装上の注記**: アカウント作成時、`is_verified=false`で作成すると同時に確認用トークンを発行し、確認リンク付きのメールを送信する(N-006、F-037、2026-07-13追加)。メール送信に失敗してもエラーレスポンスは返さず、登録処理自体は継続する(既存のN-001〜N-005と同じパターン)
