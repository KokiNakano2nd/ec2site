# PATCH /admin/orders/{order_id}/return — 返品を承認・却下する(管理者)

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 注文管理業務(管理者向け) / **リソース**: /admin/orders

**認証**: 必要(Bearerトークン + 管理者権限)

**概要**: 返品申請中(`return_requested`)の注文を承認または却下する。承認時は在庫・クーポン使用回数を戻しStripe決済済みの場合は返金する。却下時は`shipped`に戻す(S-103)
**元になった機能**: F-033
**関連する通知**: `04_notification_design.md` N-002(承認時)、N-004(却下時、2026-07-12追加)

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| order_id | integer(パスパラメータ) | 必須 | 注文ID |
| action | string | 必須 | `approve`(承認) または `reject`(却下) |

**レスポンス(200)**: 注文オブジェクト(承認時は`returned`、却下時は`shipped`)

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 404 | 注文が存在しない | 「注文が見つかりません」 |
| 400 | ステータスが`return_requested`以外 | 「この注文は返品申請中ではありません」 |
| 400 | `action`が`approve`/`reject`以外 | 「actionはapproveまたはrejectを指定してください」 |
| 500 | Stripe返金APIの呼び出しに失敗(承認時) | 「返金処理に失敗しました」 |
