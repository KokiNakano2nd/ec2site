# POST /orders/{order_id}/cancel — 注文をキャンセルする

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 商品購入業務 / **リソース**: /orders

**概要**: 発送前(`pending`/`processing`)の自分の注文をキャンセルする。在庫・クーポン使用回数を戻し、Stripe決済済みの場合は返金する(S-004)
**元になった機能**: F-031
**認証**: 必要(Bearerトークン)
**関連する通知**: `04_notification_design.md` N-002

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| order_id | integer(パスパラメータ) | 必須 | 注文ID |

**レスポンス(200)**: 注文オブジェクト(ステータス`cancelled`)

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 404 | 注文が存在しない、または当該顧客のものでない | 「注文が見つかりません」 |
| 400 | ステータスが`pending`/`processing`以外 | 「発送済みの注文はキャンセルできません」 |
| 500 | Stripe返金APIの呼び出しに失敗 | 「返金処理に失敗しました」 |

**実装上の注記**: `stripe_payment_intent_id`が設定されている場合のみStripe返金APIを呼び出す(Stripeを使わず確定した注文の返金は運用側の手動対応、UC-006備考参照)
