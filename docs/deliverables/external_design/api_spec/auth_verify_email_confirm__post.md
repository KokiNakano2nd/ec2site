# POST /auth/verify-email/confirm — メールアドレスを確認する

[← API仕様一覧に戻る](../02_api_spec.md)

**業務領域**: 会員管理業務 / **リソース**: /auth

**概要**: 確認用トークンを検証し、メールアドレスを確認済み状態にする(確認メール内のリンクから自動的に呼び出される)
**元になった機能**: F-037
**認証**: 不要(確認トークン自体が本人確認の代わりとなる)

**リクエスト**:
| 項目名 | 型 | 必須 | 説明 |
|---|---|---|---|
| token | string | 必須 | 確認メール内のリンクに含まれるトークン |

**レスポンス(200)**: 本文なし

**エラーレスポンス**:
| ステータスコード | 条件 | メッセージ |
|---|---|---|
| 400 | トークンが存在しない、既に使用済み、または発行から7日を超えて期限切れ | 「リンクが無効です。再度お手続きください」 |

**実装上の注記**: 成功時、`is_verified`を`true`に更新し、`email_verification_token`・`email_verification_token_expires_at`を`NULL`にクリアすることでトークンを再利用不能にする(1回限りの使用制限、UC-010参照)。フロントエンドはフォーム入力を介さず、確認リンク(`?verify_token=...`)を開いた時点で自動的に本APIを呼び出す(UC-010備考参照)
