# UC-004: 商品にレビューを投稿する

[← ユースケース一覧に戻る](../01_use_cases.md)

**業務領域**: レビュー投稿業務

**元になったUser Story**: US-009
**主アクター**: 顧客
**ステークホルダーと関心事**:
- 顧客: 自分の体験を正確に共有したい。他の顧客のレビューを購入判断の参考にしたい
- 運営: 1顧客が同じ商品に何度も投稿して評価を操作できないようにしたい

**事前条件**: 顧客がログイン済みであり、対象商品が存在する
**事後条件(成功時)**: レビュー(評価・コメント)が登録され、商品詳細画面のレビュー一覧に反映される

**基本フロー**:
1. 顧客が、商品詳細画面のレビューセクションで評価(1〜5)とコメント(任意)を入力する
2. 顧客が、「投稿する」ボタンを押す
3. システムが、対象商品が存在することを確認する
4. システムが、評価が1〜5の範囲内であることを確認する
5. システムが、この顧客がこの商品に未投稿であることを確認する
6. システムが、レビューを登録する
7. システムが、商品詳細画面のレビュー一覧の先頭に新しいレビューを表示する

**代替フロー**: なし

**例外フロー**:
- 3a. 対象商品が存在しない場合
  1. システムが、「商品が見つかりません」エラーを表示する
- 4a. 評価が1〜5の範囲外の場合
  1. システムが、「評価は1〜5で指定してください」エラーを表示する
- 5a. この顧客がこの商品に既に投稿済みの場合
  1. システムが、「すでにレビューを投稿済みです」エラーを表示する

**関連する概念エンティティ**: 顧客, 商品, レビュー
**関連する画面**: 商品一覧画面(商品詳細)
