# UC-001: クーポンを適用する

[← ユースケース一覧に戻る](../01_use_cases.md)

**業務領域**: 商品購入業務

**元になったUser Story**: US-002
**主アクター**: 顧客
**ステークホルダーと関心事**:
- 顧客: 正しいクーポンコードであれば確実に割引を受けたい
- 運営: 無効化されたクーポンや使用回数上限を超えたクーポンが適用されないようにしたい(不正利用の防止)

**事前条件**: カートに商品が1つ以上入っている
**事後条件(成功時)**: カートの表示金額に、クーポンによる割引が反映されている

**基本フロー**:
1. 顧客が、カート画面のクーポンコード入力欄にコードを入力する
2. 顧客が、適用ボタンを押す
3. システムが、入力されたコードに一致する有効(`is_active`)なクーポンを検索する
4. システムが、クーポンの使用回数が上限(`max_uses`)未満かを判定する
5. システムが、クーポンの割引タイプ(定率/定額)に応じて割引額を計算する
6. システムが、カート画面に適用済みクーポンと割引後の金額を表示する

**代替フロー**:
- 3a. 一致する有効なクーポンが見つからない場合
  1. システムが、「無効なクーポンコードです」エラーを表示する
- 4a. クーポンの使用回数が上限に達している場合
  1. システムが、「このクーポンは使用回数の上限に達しています」エラーを表示する

**例外フロー**: なし(本ユースケースでは通信エラー等の技術的例外は非機能要件側で扱う)

**関連する概念エンティティ**: 顧客, カート, クーポン
**関連する画面**: カート画面
