# 非機能要件

## 1. 文書方針

IPA「非機能要求グレード」の6大項目を分類の基礎とし、品質特性の観点はISO/IEC 25010を参照する。各要件には検証方法と現状を記載する。`適合`は検証済み、`部分適合`は一部のみ、`未適合`は要求に対して実装不足、`未測定`は実装の有無にかかわらず客観的測定がない状態である。

本プロジェクトは現在ローカル開発・CI用途であり、本番サービスではない。したがって、事業SLAを実績や根拠なく確定値として記載しない。本番化に必要な項目は「本番化ゲート」として明示する。

`NFR-023`は欠番である。既存参照を壊さないため、後続IDを詰めたり欠番を別要件へ再利用したりしない。

## 9. 環境・エコロジー

現時点では測定対象となる運用基盤がなく対象外とする。本番クラウド選定時に、利用率、不要環境停止、ログ/バックアップ保持によるストレージ量を運用設計で検討する。
