# システム要求仕様

## 1. 文書の目的

本書はTechStore ECサイト全体の目的、境界、スコープ、制約、前提、未決事項を定義する。個別の要求はUser Story、ユースケース、機能要件、非機能要件を参照する。

本プロジェクトには実在する発注者・業務エキスパートがいないため、業務上の内容はユーザーが明示的に確認したものを除き「学習用の仮説」である。実装から発見した振る舞いは、業務要求として自動的に承認せず、`実装由来・未確認`として扱う。

## 2. システムの目的

| ID | 目的 | 状態 | 検証方法 |
|---|---|---|---|
| OBJ-001 | 顧客が商品を閲覧し、カート、クーポン、注文、決済をオンラインで完結できる | 学習用スコープとして確認済み | 対応US/UC/FとE2Eテスト |
| OBJ-002 | 顧客が会員情報、配送先、お気に入り、レビュー、注文後操作を管理できる | 学習用スコープとして確認済み | 対応US/UC/FとE2Eテスト |
| OBJ-003 | 管理者が商品、画像、クーポン、注文、返品、売上状況を管理できる | 学習用スコープとして確認済み | 対応US/UC/FとE2Eテスト |
| OBJ-004 | FastAPI、React、SQLAlchemy、自動テスト、CI、設計文書を通じてWeb開発工程を学習する | プロジェクト目的として確認済み | リポジトリ成果物とCI |

売上、転換率、利用者数等の事業KPIは、実運用する事業ではないため現時点で設定しない。将来本番サービス化する場合は、OBJとは別に事業目標とKPIを定義する。

## 3. ステークホルダーとシステム境界

| 区分 | 対象 | システムとの関係 |
|---|---|---|
| 人 | 顧客 | ブラウザから顧客向け機能を利用する |
| 人 | 管理者 | ブラウザから管理者向け機能を利用する |
| 外部システム | Stripe | Checkoutと返金を提供する。カード情報は本システム境界外 |
| 外部システム | SMTPサーバー | トランザクションメールを配送する |
| 開発基盤 | GitHub Actions/Dependabot/CodeQL | CI、依存更新、静的解析を提供する。実行時システム境界外 |

「業務エキスパート」は現在存在しない。要求確認先が必要な項目は、プロジェクト所有者が意思決定者となる。

## 4. スコープ

### 4.1 スコープ内

- 商品閲覧、関連商品、商品画像、カート、クーポン
- 会員登録、ログイン、退会、パスワードリセット、メールアドレス確認
- 配送先、お気に入り、レビュー
- Stripe Checkoutおよびカードを使わない注文確定
- 注文履歴、キャンセル、返品申請・承認/却下、返金
- 管理者の商品・画像・クーポン・注文管理、売上分析、残数アラート
- メール通知
- ローカル開発、テスト、CIおよび設計文書

### 4.2 現在のスコープ外

| 項目 | 理由 | 再検討条件 |
|---|---|---|
| 実商用サービスとしての公開・SLA保証 | ホスティング・運用主体未決定 | 本番公開を決めた時点 |
| 倉庫・配送業者API連携 | 学習スコープ未定義 | 実配送を扱う時点 |
| 複数税率、請求書、会計連携 | 業務要件未確認 | 実販売または会計要件発生時 |
| 多言語・多通貨 | 対象利用者未定義 | 日本国外を対象にする時点 |
| 高可用クラスタ・水平スケール | 現行は単一プロセス/SQLite | 可用性・負荷目標を設定した時点 |

「本番公開前に必要」な設計をスコープ外と書いて永久に見送らない。デプロイ設計、セキュリティ、バックアップ、監視、Stripe Webhookは本番化ゲートとして別文書で管理する。

## 5. 制約と前提

| ID | 内容 | 状態 |
|---|---|---|
| CON-001 | フロントエンドはReact/ViteのSPA、バックエンドはFastAPI | 採用済み技術制約 |
| CON-002 | 開発・テストのDBはSQLite。商用DBは未選定 | 現状/未決定 |
| CON-003 | カード情報を保持せずStripe Checkoutへ委任する | 確認済み設計制約 |
| CON-004 | 現行認証は60分有効のJWT Bearer方式 | 実装済み・セキュリティ再評価対象 |
| CON-005 | 金額・割引・税の現行実装は`float`と固定税率10% | 実装由来・業務未確認・移行対象 |
| CON-006 | メール未確認でも購入を許可する | 実装由来・業務未確認 |
| CON-007 | カードを使わない注文の実際の支払方法・回収業務は未定義 | 未決事項 |
